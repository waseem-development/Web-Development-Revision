What are these 3 imports?

Breaking down import { Client, Account, ID } from "appwrite"

Think of "appwrite" as a toolbox. When you run npm install appwrite, you get a big box full of tools. These three imports are the specific tools you pull out of that box for this auth file.
🔌

Client

The Connection

Client is like your phone — the base device that connects you to Appwrite's servers. Before anything works, you tell it where Appwrite lives (endpoint) and which project you're using (project ID).

const client = new Client();
client.setEndpoint("https://...");
client.setProject("abc123");
👤

Account

The Auth Worker

Account is the specific tool for authentication. It needs the Client to work. Once you give it the Client, it unlocks all auth methods: create accounts, login, logout, verify email, reset password.

const account = new Account(client);
// Now you can use:
account.create(...)
account.createSession(...)
🎲

ID

The ID Generator

ID is a simple utility helper with one job: generate a unique random ID. Every user, token, and document in Appwrite needs a unique ID. Instead of making one yourself, ID.unique() handles it instantly.

ID.unique()
// → "64f3a2b9c8e1d" (random)
// Different every single time
auth.js — the three imports explained
// "appwrite" is the npm package you installed
// You're pulling out 3 specific tools from it:

import { Client, Account, ID } from "appwrite";

// Client  → sets up the CONNECTION to Appwrite servers
//           like a phone that knows who to call

// Account → gives you all AUTH methods (login, signup etc.)
//           but NEEDS Client to work (needs the phone first)

// ID      → generates a random unique ID string
//           ID.unique() → "abc123xyz789" (different every time)

// conf is YOUR config file — holds your Appwrite project details
import conf from "../conf/conf";
Why import only these 3? The appwrite package has many tools — Databases, Storage, Functions, etc. You only import what you actually use, keeping your code clean and your bundle small.

Why a Class? Why an Object?

Understanding export class AuthService and const authService = new AuthService()

Simple answer: A class is a blueprint. An object is the real thing built from that blueprint. You write the blueprint once, then build one real thing from it to use everywhere in your app.
The Blueprint
class AuthService
Just a description. Defines what properties and methods the thing will have. Nothing actually runs yet.
→
The Real Thing
new AuthService()
Actually creates it in memory. Now you can call .login(), .logout() etc. on it.
📐
Why write a Class at all? export class AuthService
A class lets you group all auth-related logic in one place. Instead of having login(), logout(), createAccount() scattered across different files, they all live together inside AuthService. This is called encapsulation.
🍔 Like a restaurant menu — all food options are grouped in one place. You don't hunt around to find what's available.
🏭
Why new AuthService()? creating an object
The class is just a description — it doesn't DO anything until you use new to create a real instance of it. new AuthService() runs the constructor, sets up the Appwrite client, and gives you a ready-to-use object.
🏠 The class is the architect's floor plan. new AuthService() is actually building the house. You can't live in a floor plan — you need the real house.
🌍
Why export the object, not the class? export default authService
We create one object and export that. Every file that imports it shares the exact same auth instance. This pattern is called a Singleton — one shared instance for the whole app. No wasted memory, no setup duplication.
🔑 Like having one master key for the whole building. Every room (file) uses the same key — you don't make a new one for each room.
The class → object pattern annotated
// ── STEP 1: Define the blueprint ─────────────────────────────
export class AuthService {
  // all auth methods live here grouped together
  async login() { ... }
  async logout() { ... }
}
// ↑ Nothing has run yet. This is just a recipe.


// ── STEP 2: Build the real thing (instance/object) ───────────
const authService = new AuthService();
// ↑ NOW it runs. Constructor fires, client gets configured.


// ── STEP 3: Export the single shared instance ────────────────
export default authService;
// ↑ Every file that imports this gets THE SAME object.
//   Not a new copy — the exact same one. (Singleton pattern)


// ── STEP 4: Use it anywhere ──────────────────────────────────
import authService from "./auth";
authService.login({ email, password }); // ✅
authService.logout();                   // ✅

What is a Constructor?

Understanding the constructor() method and why it exists

One sentence: A constructor is a special function that automatically runs once the moment you write new AuthService() — it sets everything up so the object is ready to use immediately.
🚗
Real world analogy
Think of a car factory. The class is the car model blueprint. new Car() is the moment a car rolls off the assembly line. The constructor is the assembly line — it runs automatically to put all parts together so the car is ready to drive when it comes out.
🏭 You don't manually attach the engine after buying a car. The factory (constructor) already did it before handing it to you.
1
You write new AuthService()
JavaScript sees the new keyword and immediately jumps inside the constructor() function.
const authService = new AuthService(); ← constructor fires here
2
A blank Client is created
client = new Client() is declared as a class field. It creates a blank Appwrite client — like a phone with no SIM card yet.
client = new Client(); ← blank, no config yet
3
Client gets configured inside the constructor
The constructor calls .setEndpoint() and .setProject() — like inserting the SIM card. Now the client knows WHERE Appwrite is and WHICH project to connect to.
this.client
.setEndpoint(conf.appwriteEndpoint) ← "where is Appwrite?"
.setProject(conf.appwriteProjectId); ← "which project?"
4
Account is created using the configured Client
Account needs a client to know which server to talk to. We pass this.client to it. Now this.account is fully ready with all auth methods available.
this.account = new Account(this.client);
// ↑ account now knows which Appwrite server to talk to
5
Constructor finishes — object is fully ready
By the time new AuthService() returns, everything is set up. You can immediately call any method on it.
// authService is fully ready:
authService.login({ email, password }); ✅
authService.logout(); ✅
auth.js — constructor fully annotated
export class AuthService {

  // Class fields — set up BEFORE constructor body runs
  client = new Client();  ← blank client (no config yet)
  account;                 ← declared but empty (undefined) for now

  // constructor() — automatically called by: new AuthService()
  constructor() {

    // "this" = the specific object being created right now
    this.client
      .setEndpoint(conf.appwriteEndpoint)  ← WHERE is Appwrite?
      .setProject(conf.appwriteProjectId);  ← WHICH project?

    // Pass the now-configured client into Account
    this.account = new Account(this.client); ← account is ready!

  } // ← constructor ends. Object is 100% ready to use.

  // All methods below can safely use this.account
  async login({ email, password }) {
    return await this.account.createSession({ email, password });
  }
}
What is this? Inside a class, this refers to the specific object that was just created. this.client means "the client that belongs to THIS authService object". this.account means "the account that belongs to THIS authService object". It's how the object refers to itself.

auth.js → createAccount()
async createAccount({ email, password, name }) {
  try {
    // Step 1: Create user in Appwrite's database
    const userAccount = await this.account.create(
      ID.unique(),  // auto-generates "64f3a9b2c1" — never repeated
      email,        // "john@example.com"
      password,     // "mypassword123"
      name          // "John Doe"
    );
    // Step 2: If success, immediately log them in
    if (userAccount) {
      return this.login({ email, password }); // calls login() below
    }
  } catch (error) {
    console.error("Appwrite Error :: createAccount", error);
    throw error;
  }
}

What is a Session? createSession()
A session is temporary proof that you're logged in. Appwrite stores it and checks it every time you do something protected.
🎫 Like a concert wristband — while you wear it you can come and go. Removed = you're out.
🍪
Where is the session stored?
Appwrite automatically stores it as a secure cookie in the browser. You don't manage tokens manually.
🧠 The browser remembers you're logged in, so refreshing doesn't log you out.
auth.js → login()
async login({ email, password }) {
  try {
    return await this.account.createSession({ email, password });
    //                         ↑ new in SDK v16+
    // Old (deprecated): createEmailPasswordSession(email, password)
  } catch (error) {
    console.error("Appwrite Error :: login", error);
    throw error;
  }
}

🔍
Checks active session account.get()
Every time your app loads, call this to check if the user is already logged in. Appwrite reads the cookie and returns user data.
📱 Like how Instagram shows your feed when you reopen the app — no login prompt needed.
❌
No session? → return null
If the session expired or user never logged in, account.get() throws an error. We catch it and return null — meaning nobody is logged in.
🔒 Like an expired hotel keycard — the door won't open, you need a new one (log in again).
auth.js → getCurrentUser()
async getCurrentUser() {
  try {
    return await this.account.get();
    // Returns: { $id, name, email, emailVerification, ... }
  } catch (error) {
    console.error("Appwrite Error :: getCurrentUser", error);
    return null; // ← return null, don't throw — no session is fine
  }
}

// Usage in React (App.jsx):
const user = await authService.getCurrentUser();

📄
Complete auth.js

The entire file — clean, annotated, no deprecated methods

src/appwrite/auth.js
import conf from "../conf/conf";
import { Client, Account, ID } from "appwrite";
// Client  → connection to Appwrite servers
// Account → all auth methods
// ID      → generates unique IDs

export class AuthService {             // Blueprint
  client = new Client();              // blank connection
  account;                            // filled in constructor

  constructor() {                     // runs on: new AuthService()
    this.client
      .setEndpoint(conf.appwriteEndpoint)
      .setProject(conf.appwriteProjectId);
    this.account = new Account(this.client); // ready!
  }

  // ── Create Account ──────────────────────────────────────────
  async createAccount({ email, password, name }) {
    try {
      const userAccount = await this.account.create(ID.unique(), email, password, name);
      if (userAccount) return this.login({ email, password });
      return userAccount;
    } catch (error) { console.error("Appwrite Error :: createAccount", error); throw error; }
  }

  // ── Login ────────────────────────────────────────────────────
  async login({ email, password }) {
    try {
      return await this.account.createSession({ email, password });
    } catch (error) { console.error("Appwrite Error :: login", error); throw error; }
  }

  // ── Get Current User ─────────────────────────────────────────
  async getCurrentUser() {
    try {
      return await this.account.get();
    } catch (error) { console.error("Appwrite Error :: getCurrentUser", error); return null; }
  }

  // ── Logout ───────────────────────────────────────────────────
  async logout() {
    try {
      await this.account.deleteSessions();
    } catch (error) { console.error("Appwrite Error :: logout", error); }
  }

  // ── Send Email Verification OTP ──────────────────────────────
  async sendVerification() {
    try {
      const user = await this.account.get();
      return await this.account.createEmailToken(ID.unique(), user.email);
    } catch (error) { console.error("Appwrite Error :: sendVerification", error); throw error; }
  }

  // ── Confirm Email Verification ───────────────────────────────
  async confirmVerification(userId, secret) {
    try {
      return await this.account.createSession({ userId, secret });
    } catch (error) { console.error("Appwrite Error :: confirmVerification", error); throw error; }
  }

  // ── Send Password Recovery OTP ───────────────────────────────
  async passwordRecovery({ email }) {
    try {
      return await this.account.createEmailToken(ID.unique(), email, true);
    } catch (error) { console.error("Appwrite Error :: passwordRecovery", error); throw error; }
  }

  // ── Confirm Password Recovery ────────────────────────────────
  async confirmPasswordRecovery(userId, secret, newPassword) {
    try {
      await this.account.createSession({ userId, secret });
      return await this.account.updatePassword(newPassword);
    } catch (error) { console.error("Appwrite Error :: confirmPasswordRecovery", error); throw error; }
  }
}

const authService = new AuthService(); // ← ONE shared instance (Singleton)
export default authService;            // ← whole app uses this same object
